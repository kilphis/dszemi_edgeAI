# なぜ「四角形」ではなく「多角形」でゾーンを定義するのか

## 問題：カメラが斜めだと四角形がズレる

カメラが駐輪場を真上から撮っていれば、禁止エリアを「左上の座標・右下の座標」の四角形で定義するだけで十分です。

でも今回のカメラは斜めから撮影しています。
すると、現実の空間では「まっすぐな通路」でも、画像の上では**台形**に見えます。

```
現実の通路（上から見た図）        カメラ画像（斜めから見た図）

 ┌──────────────────────┐          ／￣￣￣￣￣￣￣￣￣＼
 │                      │    →   ／                     ＼
 └──────────────────────┘        ＼_______________________／
       長方形                            台形
```

四角形ゾーンでこれを無理やり表現しようとすると、**本当は合法なエリアまで禁止ゾーンに含まれてしまう**（誤検知が増える）という問題が起きます。

---

## 解決策：頂点のリストで形を定義する（ポリゴン）

多角形（ポリゴン）は、頂点の座標をいくつでも並べることで、台形でも五角形でも自由な形を作れます。

```python
# 四角形（矩形）による定義 → 斜め映像に弱い
{"name": "aisle", "X": 0, "Y": 240, "x": 320, "y": 320}

# ポリゴンによる定義 → 台形にできる
{"name": "aisle", "polygon": [(0, 230), (320, 270), (320, 320), (0, 320)]}
```

台形の4頂点を時計回りに並べるだけで、実際の通路の形にぴったり合わせられます。

## 「点がゾーン内にあるか」の判定：レイキャスティング法

自転車の検出結果は「重心の座標 (cx, cy)」として得られます。
この点がポリゴンの内側にあるかどうかを判定するアルゴリズムが**レイキャスティング法**です。

### アイデア

その点から右向きに光線（レイ）を飛ばします。
光線がポリゴンの辺と**奇数回**交われば内側、**偶数回**なら外側です。

```
          レイ →→→→→→→→
     ×           ╱
  (外側)    ╱ ポリゴン ╲        ×
       ────╱             ╲──  (外側)
          │               │
    ・    │    ・          │
 (外側)   │  (内側)        │
          ╲               ╱
           ╲_____________╱

  内側の点から飛ばしたレイ → 辺と1回交差（奇数）→ 内側 ✓
  外側の点から飛ばしたレイ → 辺と0回か2回交差（偶数）→ 外側 ✗
```

### コード（17行）

```python
def point_in_polygon(px, py, polygon):
    inside = False
    n = len(polygon)
    j = n - 1
    for i in range(n):
        xi, yi = polygon[i]
        xj, yj = polygon[j]
        if ((yi > py) != (yj > py)) and \
           (px < (xj - xi) * (py - yi) / (yj - yi) + xi):
            inside = not inside
        j = i
    return inside
```

辺ごとに「レイと交差するか」を調べて、交差のたびに `inside` を反転させています。
ループが終わった時点で `True` なら内側、`False` なら外側です。

## まとめ

| | 矩形ゾーン | ポリゴンゾーン |
|---|---|---|
| 定義の方法 | 左上・右下の2点 | 頂点のリスト（何点でも） |
| 対応できる形 | 長方形のみ | 台形・五角形・任意の多角形 |
| 斜めカメラへの対応 | ✗ ズレが生じる | ✓ 形に合わせられる |
| 判定アルゴリズム | 矩形の重なり計算（IoU） | レイキャスティング法 |
| 実装の複雑さ | シンプル | やや増えるが17行で書ける |

斜め撮影のカメラで正確なゾーン判定をするために、このプロジェクトではポリゴンゾーンを採用しました。
